开发流程

功能点梳理

引言:

从 Element、AntD 、Material-UI等优秀组件站点中，可以看到组件门户具备的一些特质：

  • 分类清晰，便于检索
  • 案例case比较具体和丰富，可覆盖多数应用场景
  • 以表格形式聚合API信息
  • 支持codepen等在线编辑器调试
  • 较为详细的组件使用文档

综上，针对整站主体功能页面和单个组件页面，这两个不同的方面进行功能点梳理。

1、整站主体页面

首页、顶部导航、侧边栏导航

1.1 首页

建议首页直接放快速入门，内容包括：

  • 简单介绍组件库
  • 安装、使用 、举一个简单的例子
1.2 顶部导航
  • 搜索框
  • 设计语言
  • github (链接到组件库的github)
  • 版本列表
1.3 侧边栏导航
  • 贡献指南（开发过程中，大家都有可能贡献组件，因此"贡献指南"比较重要）
    • 项目如何启动
    • 代码规范
    • 如何开发组价
    • 如何发布组件
  • 更新日志
    • Features 每次版本更新了什么
    • Bug Fixes修复了什么
  • 组件列表
    • 按照一定标准对组件进行分类

2、单个组件页面

功能描述、使用案例、接口说明

  • 功能描述: 说明这个组件的功能、适用场景（简单说明就好）
  • 使用案例: AntD、Element、Material-UI 的使用案例基本覆盖了常用的需求场景 对于我们的组件库，最开始的时候，覆盖最常用的场景就好，逐步迭代优化
  • 接口说明: 重要/常用的 API ，建议进行比较详细的说明，让大家都看懂，以表格的形式呈现

参考：

  1. Ant Design
  2. Element
  3. Material-UI
  4. [云音乐MUI组件库] (http://mui.igame.163.com/)
